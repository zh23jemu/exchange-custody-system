# 换汇资金托管系统需求简述

## 一、项目目标

开发一套基于 Python 的 Web 系统，用于管理换汇业务中的客户资金流转过程，帮助操作人员清晰记录每笔资金从入账、托管、换汇到转出的全过程。

系统重点解决以下问题：

- 记录客户资金是如何进入公司账号的
- 跟踪资金当前处于哪个处理阶段
- 管理公司账号多币种余额
- 记录换汇过程和换汇结果
- 管理客户最终收款账号
- 生成客户资金流水，方便对账

---

## 二、使用场景

适用于小型换汇、资金代管、钱庄类业务场景，例如：

- 客户交现金，由中转商代为转入公司账号
- 客户直接银行转账到公司账号
- 公司收到资金后，按客户要求换汇
- 公司最终将资金转到客户指定收款账号

---

## 三、核心功能

### 1. 实体管理

系统需要支持维护以下基础资料：

- 客户
- 中转商
- 公司账号
- 客户目标账号

每项资料至少支持新增和查看。

### 2. 订单管理

系统需要支持两类订单：

- 现金订单
- 直接转账订单

每笔订单需记录：

- 客户
- 中转商（现金订单需要）
- 公司账号
- 入账金额与币种
- 转出金额与币种
- 当前状态
- 备注

说明：

- 客户资金进来时，不需要立即确认转出目标账号
- 创建订单时，客户目标账号可以留空
- 创建订单时，转出金额和转出币种也可以留空
- 目标账号可以在后续准备转出时再补充
- 转出金额和转出币种可以在后续准备转出时再补充
- 但在订单完成转出前，必须补齐目标账号
- 在订单完成转出前，必须补齐转出金额和转出币种

### 3. 订单状态流转

现金订单流程：

- 待处理
- 交中转商
- 在公司账号
- 已完成

直接转账订单流程：

- 在公司账号
- 已完成

系统需支持订单按规则推进状态，不允许随意跳转。

### 4. 资金台账与余额

系统需要自动记录公司账号的资金变动，包括：

- 订单入账
- 订单转出
- 货币兑换扣减
- 货币兑换增加

并根据台账实时计算各公司账号在不同币种下的余额。

### 5. 货币兑换

系统需要支持公司账号内部的多币种兑换，并记录：

- 源币种
- 源金额
- 目标币种
- 目标金额
- 实际汇率
- 兑换时间

若源币种余额不足，不允许兑换。

### 6. 汇率设置

系统需要支持维护汇率设置，采用：

- `1 AUD = ? 目标币种`

作为基础汇率规则，用于不同币种之间的换算。

### 7. 客户流水

系统需要按客户查看完整资金流水，包括：

- 入账记录
- 转出记录
- 按币种汇总净变化
- 客户已登记的目标账号

---

## 四、技术要求

- 主体程序使用 Python
- 系统采用 Web 方式实现
- 后端框架使用 FastAPI
- 页面使用服务端模板渲染
- 数据库存储使用 SQLite
- 适合单机部署或轻量服务器部署

---

## 五、当前版本范围

当前版本主要实现：

- 基础资料管理
- 订单创建与状态推进
- 账号余额管理
- 货币兑换
- 客户流水查看
- 汇率设置
- 示例数据初始化

当前版本暂不包含：

- 登录和权限控制
- 云同步
- 导入导出
- 订单撤销/回退
- 多用户协作

---

## 六、页面结构

系统主要页面包括：

- 首页总览
- 订单管理
- 实体管理
- 账号余额
- 货币兑换
- 客户流水
- 汇率设置

---

## 七、交付目标

交付一个可直接运行的本地 Web 系统，满足业务人员对换汇资金流、订单状态、账号余额和客户流水的日常管理需求。
